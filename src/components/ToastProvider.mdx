# ToastProvider

Manages a queue of toast notifications with auto-dismiss, stacking, and manual dismissal.

## Setup

Wrap your app (or the relevant subtree) once at the root:

```tsx
import { ToastProvider } from './components/ToastProvider'

function App() {
  return (
    <ToastProvider>
      <YourApp />
    </ToastProvider>
  )
}
```

## useToast()

Call `useToast()` anywhere inside the provider tree to get `add` and `dismiss`.

```tsx
import { useToast } from './components/ToastProvider'

function SaveButton() {
  const { add } = useToast()

  const handleSave = async () => {
    await save()
    add({ variant: 'success', message: 'Changes saved!' })
  }

  return <button onClick={handleSave}>Save</button>
}
```

### `add(toast)`

| Param | Type | Required | Default | Description |
|---|---|---|---|---|
| `variant` | `'success' \| 'error' \| 'warning' \| 'info'` | Yes | — | Visual style |
| `message` | `string` | Yes | — | Notification text |
| `duration` | `number` (ms) | No | `4000` | Auto-dismiss delay. `0` = never dismiss |

Returns a `string` ID you can pass to `dismiss()` if you need to remove it programmatically.

### `dismiss(id)`

Removes the toast with the given ID immediately. Safe to call with a non-existent ID (no-op).
